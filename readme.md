# 数据结构与算法学习笔记
## 时间复杂度
### 概念
时间复杂度代表的是随着数据量增长，代码执行时间的变化趋势
### 分析方法

#### 只关注循环执行次数最多的一段代码
我们在分析一个算法、一段代码的时间复杂度的时候，也只关注循环执行次数最多的那一段代码就可以了
####  加法法则：总复杂度等于量级最大的那段代码的复杂度
O(1)+O(n)+O(n**2)=O(n**2)
#### 乘法法则：嵌套代码的复杂度等于嵌套内外代码复杂度的乘积
O(n)*O(n)=O(n**2)

### 时间复杂度复杂度量级分类
#### 多项式量级
常量阶O(1)</br>
对数阶O(logn)</br>
线性阶O(n)</br>
线性对数阶O(nlogn)</br>
平方阶O(n**2) O(n**3).....
#### 非多项式量级
指数阶O(2**n)
阶乘阶O(n!)
非多项式时间复杂度的算法其实是非常低效的算法,并不常用
## 空间复杂度
表示算法的执行时间与数据规模之间的增长关系
